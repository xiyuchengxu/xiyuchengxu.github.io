# 多页面导航与精简首页设计

## 目标

将当前把搜索、标签、归档和关于内容混在首页的单页信息流，调整为独立页面导航。首页在移动端只展示文章流；桌面端保留简短右侧摘要栏，但所有摘要项目均跳转至专用页面。整体交互参考 X 的纯图标移动底部导航密度，不使用 X 的品牌资产、文案或图标资源。

## 范围

本次变更只调整 `docs/` 下的静态展示层和前端测试。

- 保留：Markdown 文章源、Python 构建器、`docs/posts.js` 数据格式、独立文章页和 GitHub Actions 构建链。
- 新增：搜索、归档、标签和关于的独立 HTML 页面。
- 修改：首页结构、共享前端脚本、共享样式、静态契约测试和 Playwright 截图测试。
- 不包含：评论、登录、文章后台、外部图标库、前端框架或新的线上服务。

## 信息架构

### 页面与 URL

| 页面 | 路径 | 职责 |
|---|---|---|
| 首页 | `docs/index.html` | 仅渲染文章信息流、骨架屏和桌面右侧摘要入口。 |
| 搜索 | `docs/search.html` | 默认聚焦搜索框，按标题、摘要和标签即时筛选文章。 |
| 归档 | `docs/archive.html` | 按 `YYYY-MM` 倒序分组文章，显示文章链接。 |
| 标签 | `docs/tags.html` | 显示标签与文章数量；选定标签后在本页显示匹配文章和清除操作。 |
| 关于 | `docs/about.html` | 展示固定站点简介、作者身份和联系方式，不渲染文章流。 |

文章详情 URL 继续由构建器生成，例如 `docs/posts/2024-01-15-hello-world.html`。

### 导航

所有页面共享首页、搜索、归档、标签、关于五个导航目标。

- 移动端使用固定底栏，严格只显示五个图标，不显示视觉文字标签。
- 每个移动导航链接具有 `aria-label`、`title` 和当前页的 `aria-current="page"`。
- 桌面侧栏保留图标和文字，并用左侧强调条和 `aria-current="page"` 表示当前页。
- 页面顶部只保留当前页面标题和右侧主题切换按钮；删除首页的 `个人技术笔记` 眉题、重复站点标题及顶部搜索按钮。
- 原顶部搜索位置改为主题切换按钮。搜索入口仅指向 `search.html`。

## 页面行为

### 首页

`index.html` 中央列仅包含标题为「YuCheng 的博客」的单行顶部栏、文章骨架和文章流。首页不得包含搜索表单、主题栏目按钮、标签云、归档列表或完整关于内容。

移动端隐藏右栏。桌面端右栏仅显示：简短作者介绍与关于页链接、最多 5 篇最近文章、最多 8 个标签入口和最多 6 个月份归档入口。标签和月份均跳转至各自的独立页面，并通过 URL 查询参数预选，例如 `tags.html?tag=Python`、`archive.html?month=2024-01`。

### 搜索

`search.html` 加载后将焦点置于搜索框。文章数据加载完成前显示与文章流等高的骨架；无搜索词时显示最近文章；有搜索词时按标题、摘要、标签做不区分大小写匹配；无匹配时显示清除按钮。结果中的标签链接到 `tags.html?tag=<encoded-tag>`，文章标题链接到文章详情页。

### 归档

`archive.html` 从 `posts` 数据中按 `YYYY-MM` 倒序分组。`month` 查询参数存在时只显示该月份，并显示清除筛选入口；未指定时显示全部月份。月份标题和文章链接均为可访问的语义化列表。

### 标签

`tags.html` 从 `posts` 数据计算标签及数量，按数量倒序、名称升序展示。`tag` 查询参数存在时保持标签列表可见，并在下方渲染匹配文章；没有匹配文章时显示空状态和清除入口。标签名和 URL 查询参数必须使用 `encodeURIComponent`，避免空格和中文标签破坏链接。

### 关于

`about.html` 使用固定静态内容：站点名称、作者 YuCheng、博客用途、联系邮箱 `hello@example.com` 和回到首页的链接。页面不依赖 `posts.js`，即使文章数据加载失败也必须可阅读。

## 共享前端结构

将当前单页脚本拆成可复用的纯函数与页面初始化函数，仍使用原生 JavaScript：

- 数据函数：HTML 转义、查询归一化、文章筛选、标签统计、归档分组和文章卡片标记。
- 壳函数：渲染桌面导航、移动导航、主题按钮、桌面右栏、提示条和骨架占位。
- 页面函数：`initHomePage`、`initSearchPage`、`initArchivePage`、`initTagsPage`、`initAboutPage`。
- 页面根元素通过 `data-page` 声明页面类型；脚本在 `DOMContentLoaded` 时按该值初始化。
- 所有主题按钮调用同一 `toggleTheme`，使用既有 `localStorage` 键 `blog-theme`，深浅主题跨页面保持一致。
- 复制文章链接继续保留在文章卡片中；标签按钮改为带查询参数的链接，不能再回写首页搜索框。

## 视觉与响应式规则

- 桌面宽度 `>= 1180px`：左侧导航、中央内容列、右侧摘要栏。
- 中等宽度 `760px` 至 `1179px`：隐藏右栏，保留桌面侧栏和中央列。
- 窄屏 `< 760px`：隐藏桌面侧栏和右栏，显示单行顶部栏与固定纯图标底栏。
- 移动底栏高度固定，页面内容底部预留相应空间；文章操作行、搜索结果和空状态不得被底栏遮挡。
- 底栏图标按钮保持至少 `44px × 44px` 的命中区域；视觉文字可完全隐藏，但不得从可访问名称中移除。
- 继续使用中性深浅主题、细分隔线、可见焦点状态和 `prefers-reduced-motion` 规则；不重新引入渐变、大圆角卡片或伪社交互动。

## 错误处理与降级

- 文章页面的数据容器使用 `aria-live="polite"` 和 `aria-busy`；数据缺失或为空时显示明确空状态。
- 搜索、标签和归档对未知查询参数显示空状态与清除入口，不抛出 JavaScript 异常。
- 每个依赖 `posts.js` 的页面提供 `noscript` 文章索引链接；关于页不依赖 JavaScript。
- 每个 HTML 页面加载 `styles.css`；依赖文章数据的页面在 `script.js` 前加载 `posts.js`。
