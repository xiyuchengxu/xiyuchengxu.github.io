# X 风格博客界面升级设计

**状态：** 已获用户确认，实施中

## 目标

将博客首页从演示型卡片布局升级为适合技术写作的高密度信息流。设计借鉴 X 的导航节奏、内容分隔、身份呈现和加载反馈，但不复制 X 的商标、文案、图形资产或社交互动数据。

优先改善首次访问者的身份认知、文章浏览和阅读进入路径。Markdown 文章源、独立文章 HTML、构建脚本和 GitHub Actions 发布链保持不变。

## 范围

本次改造修改展示层和渐进增强逻辑：

- `docs/index.html`：语义化三栏首页和移动导航结构。
- `docs/styles.css`：深浅主题、响应式栅格、信息流、骨架屏和可访问焦点样式。
- `docs/script.js`：文章信息流渲染、标签筛选、搜索、主题切换、复制链接和骨架屏替换。
- `docs/posts.js`：由构建脚本继续生成；本次只消费其数据。
- `scripts/build.py` 和 `tests/`：仅在需要补充可选展示字段或前端契约测试时修改。

本次不做以下内容：

- 不添加账户、评论、关注、点赞、转发或伪造互动数字。
- 不加入浏览器端 Markdown 解析或新的内容源。
- 不复制 X 品牌、图标、名称或文案。
- 不改变文章 URL 规则 `posts/YYYY-MM-DD-title.html`。

## 信息架构

### 桌面端

宽屏使用三栏布局：

1. 左侧导航栏：站点名「YuCheng 的博客」，首页、文章、标签、归档、关于，以及主题切换。每项由图标和文字组成。
2. 中央文章流：固定栏标题「YuCheng 的博客」，栏目选项「全部文章」「技术笔记」「随想」，文章列表和加载/空状态。
3. 右侧辅助栏：作者身份区、标签、归档和最近文章。辅助信息使用轻量分组，不堆叠为厚重的浮动卡片。

中心栏是唯一主内容区域。文章条目以细分隔线分隔，整个条目可进入独立文章详情页。

### 移动端

窄屏时改为单列：

- 顶栏显示站点名、当前栏目和主题按钮。
- 中栏展示横向可滚动的栏目或标签筛选区和文章流。
- 底部固定导航为：首页、搜索、归档、标签、关于。
- 底部导航不得遮挡最后一篇文章的操作行；文章流底部预留安全间距。

## 内容模型与文章条目

首页继续以构建生成的 `window.__POSTS__` 为唯一内容数据源。每篇文章至少消费：

- `title`
- `summary`
- `date`
- `tags`
- `url`

`readingTime` 和 `featured` 是可选的构建字段。若未提供，前端不得显示虚构值。

文章条目顺序为：作者身份和发布日期、标题、摘要、标签、真实操作行。操作行只包括阅读文章和复制链接；不显示虚构的点赞、评论、转发或浏览数。

栏目筛选根据文章标签推导：

- 「全部文章」显示全部文章。
- 「技术笔记」显示带有技术分类标签的文章。
- 「随想」显示带有随想分类标签的文章。
- 没有匹配文章时显示空状态和一个清除筛选按钮。

搜索、标签、归档和最近文章从同一份文章数据推导，不能新增独立的手工数据源。

## 视觉规范

### 颜色和层级

默认主题采用中性炭黑背景、深灰分隔线和亮蓝色操作强调；浅色主题采用白色背景、冷灰分隔线和同一蓝色强调。正文、标题和辅助元信息使用至少三档对比度，保持长文阅读舒适。

不使用大面积渐变、发光圆球、厚重阴影或大圆角卡片。文章项使用 1 px 分隔线，悬停时只改变背景色，键盘聚焦时显示可见焦点轮廓。

### 排版和密度

- 站点标题和文章标题使用清晰紧凑的无衬线字体。
- 摘要和元信息使用比标题更小的字号与更高的行高。
- 图标按钮保持稳定触控尺寸，文本在窄屏下换行或截断，不能重叠。
- 页面固定元素使用明确高度和安全区域内边距，避免内容跳动。

### 图标和动作

使用通用线性图标表示首页、搜索、归档、标签、关于、主题、复制链接和阅读。图标按钮必须有 `aria-label`；桌面端提供 tooltip；键盘可聚焦和触发。

主题切换、标签筛选、导航状态和复制成功均有即时、克制的反馈。复制成功可使用短暂提示，不能依赖阻塞式弹窗。

## 加载、空状态与无 JavaScript 降级

首次加载时，在文章流中展示 3 条与最终文章布局等宽的骨架项，包含作者行、标题、摘要和操作行的占位结构。`posts.js` 加载和渲染完成后，用真实文章替换骨架，避免布局跳动。

当 JavaScript 不可用时，首页仍展示站点身份、文章目录入口和基础导航链接，不出现空白内容区或无法使用的控件。

空状态明确说明当前筛选无结果，并提供清除筛选的操作。

## 响应式断点

- 宽屏：三栏布局，中央阅读流保持可读行宽。
- 中等屏：隐藏右侧辅助栏，保留左侧导航与中央文章流。
- 窄屏：单列布局，使用顶部栏和底部导航，筛选区域可横向滚动。

所有断点下，标题、摘要、标签、工具按钮和固定导航必须不发生遮挡或水平溢出。页面不可依赖视口宽度缩放字体。

## 可访问性

- 导航使用语义化 `nav`、列表与按钮元素。
- 纯图标操作带可访问名称和可见焦点。
- 标签筛选、主题切换、复制链接和导航均可用键盘完成。
- 深浅主题下正文、链接与辅助文本具有可读对比度。
- 遵守 `prefers-reduced-motion`，禁用非必要的骨架和状态动画。
